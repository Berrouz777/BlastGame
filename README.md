# Blast Game Prototype

Прототип игры в жанре **Blast puzzle** на **TypeScript + Cocos Creator 2.4.x**.

Live demo: https://berrouz777.github.io/BlastGame/

Репозиторий содержит исходники проекта Cocos Creator. Публичная версия для браузера собирается как `Web Mobile` и публикуется через GitHub Pages.

## Что реализовано

- Генерация игрового поля с цветными тайлами.
- Удаление группы соседних тайлов одного цвета по клику.
- Поддерживаются соседи только: вверх, вниз, влево, вправо.
- После удаления тайлы падают вниз, пустые клетки сверху заполняются новыми тайлами.
- Подсчет очков, оставшихся ходов, победы и поражения.
- Автоматическое перемешивание поля (shuffle), если больше нет доступных ходов.
- Booster Bomb.
- Booster Teleport.
- Super Tile с несколькими эффектами.
- Анимации удаления, падения, добора, перемешивания, телепорта и активации super tile.
- Разделение игровой логики и отображения.
- Unit-тесты для чистой игровой логики (без Cocos).
- Web Mobile build для запуска в мобильном браузере.

Физический движок не используется.

## Правила игры

Цель игрока — набрать нужное количество очков за ограниченное число ходов.

Текущие настройки уровня:

- Размер поля: `9 x 11`.
- Лимит ходов: `20`.
- Цель по очкам: `9000`.
- Минимальный размер группы для обычного blast: `3`.
- Максимальное количество автоматических перемешиваний: `3`.
- Формула очков: `removedTilesCount * removedTilesCount * scoreMultiplier`.
- `scoreMultiplier`: `10`.

Обычный ход:

1. Игрок кликает по тайлу.
2. Игра ищет группу тайлов такого же цвета.
3. Если размер группы меньше `3`, ход не тратится.
4. Если группа валидная, она удаляется.
5. Начисляются очки.
6. Оставшиеся тайлы падают вниз.
7. Верхние пустые клетки заполняются новыми тайлами.
8. Проверяется состояние победы, поражения и наличие доступных ходов.

Победа наступает, когда счет становится больше или равен `9000`.

Поражение наступает, если:

- ходы закончились, а цель по очкам не достигнута;
- на поле нет доступных групп, и лимит автоматических перемешиваний исчерпан.

## Дополнительные механики

### Auto Shuffle

Если после хода на поле нет доступной группы для удаления, игра автоматически перемешивает поле.

Лимит: `3` попытки.

Если после последней попытки поле всё еще не имеет доступных ходов, игра завершается поражением.

### Booster Bomb

Бустер бомбы активируется отдельной кнопкой.

Правила:

- Количество использований: `2`.
- Радиус: `1` (тоесть 1 элемент по периметру от таргет-элемента = 3х3).
- После активации следующий клик по полю удаляет тайлы в квадратной области вокруг выбранной клетки.
- Бомба тратит `1` ход.
- Очки считаются по той же формуле: `removedTilesCount^2 * 10`.
- После удаления используется общий pipeline игры: remove -> gravity -> refill -> end state check.

### Booster Teleport

Бустер телепорта позволяет поменять местами два тайла.

Правила:

- Количество использований: `3`.
- После активации игрок выбирает первый тайл, затем второй.
- Выбранные тайлы меняются местами.
- Телепорт тратит `1` ход.
- Очки за сам телепорт не начисляются.
- Нельзя телепортировать тайл сам в себя.

### Super Tile

Если обычным blast-ходом удаляется большая группа, на месте клетки, по которой кликнул игрок, создается super tile.

Текущие настройки:

- Порог создания: `>= 8` тайлов.
- `row`: `25%`.
- `column`: `25%`.
- `bomb`: `40%`.
- `clearAll`: `10%`.

Виды super tile:

- `row` — удаляет всю строку.
- `column` — удаляет весь столбец.
- `bomb` / `radius` — удаляет область радиуса `1`.
- `clearAll` — удаляет всё поле.

Активация super tile тратит `1` ход и начисляет очки за количество удаленных тайлов.

## Архитектура

Игровая логика вынесена отдельно от Cocos-представления.

Основные части:

- `assets/Script/game/model` — состояние доски и игровой сессии.
- `assets/Script/game/services` — чистые сервисы: поиск групп, гравитация, добор тайлов, проверка ходов, shuffle, resolve эффектов.
- `assets/Script/game/controller` — основной игровой controller, который связывает правила игры.
- `assets/Script/game/types` — типы доски, тайлов, результатов ходов.
- `assets/Script/board` — Cocos view/presenter слой, UI, анимации, синхронизация модели и отображения.
- `tests` — unit-тесты чистой игровой логики.

Слой game/* — чистая доменная логика (тестируемая), слой board/* — адаптер под Cocos UI/анимации. Это упрощает тестирование, развитие механик и уменьшает связность => облегчает дальнейшее расширение проекта.

## Debug / логи

В проекте **оставлены debug-логи и дополнительные проверки визуального состояния**, но они по умолчанию выключены.

Чтобы включить:

- Открыть сцену со скриптом `BoardView` в Cocos Creator.
- В инспекторе у компонента `BoardView` включить флаг `debug = true`.

## Установка

Требования:

- Cocos Creator `2.4.x` (`2.4.15` использовался при разработке).
- Node.js и npm для запуска unit-тестов.

Установка npm-зависимостей:

```bash
npm install
```

## Тесты

Запуск unit-тестов:

```bash
npm test
```

Тесты проверяют:

- поиск связанной группы;
- гравитацию;
- заполнение пустых клеток;
- наличие доступных ходов;
- shuffle;
- scoring;
- win/lose состояния;
- bomb booster;
- teleport booster;
- super tile.
